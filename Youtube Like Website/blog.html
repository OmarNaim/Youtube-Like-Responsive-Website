var

  • var has function level scope. That means it is usable throughout function.
  • var can be redecleared and reassigned.

let

  • let has block level scope. That means it is visible within the block it is declared.
  • let allows reassignment of values but not redeclaration within the same scope.

const

  • const also has block level scope.
  • It should be declared with a value and the value can not be changed.
  • It does not allow reassignment or redeclaration within the same scope.

undefined

  • Represents a variable that has been declared but has not been assigned a value.
  • It is the default value of uninitialized variables.
  • It is also returned by functions that do not explicitly return anything.
  • Often used to check if a variable has been assigned a value or not.

null

  • Represents an intentional absence of any object value.
  • It is usually assigned to variables to indicate that they don't currently refer to any object.
  • It can be used to explicitly clear the value of a variable, indicating that it no longer points to an object.
  • It means a variable has not been assigned a value yet.

Rest API

  • REST API uses HTTP methods to interact with resources.
  • Resources are represented as URLs.
  • CRUD operations (Create, Read, Update, Delete) are performed on resources.
  • It is stateless, meaning each request from the client to the server contains all the information necessary to understand and process the request.
  • Responses are typically sent in JSON or XML format, but JSON is more commonly used due to its simplicity and readability.
